# Archive

Docs in this folder are kept for historical context. They are **not**
current and should not be used to understand how Tern works today.

If something here is still useful, consider extracting the still-true part
into a current doc (under `features/`, `architecture/`, etc.) and leaving
the rest behind.

## What's here and why

- **weather-implementation-plan.md** — A plan that was explicitly marked
  COMPLETED. Kept as a record of what was delivered in the weather feature.
- **weather-comparison-ios.md** — A side-by-side comparison of weather
  features between iOS and Android. Stale because iOS development is parked
  (see `project_tern_platform_focus` in user memory).
- **weather-walkthrough.md** — A retrospective walkthrough of the weather
  parity work. Same era as the items above; useful as history, not as a
  guide.
